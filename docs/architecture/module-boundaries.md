# Module Boundaries

## Purpose

Define the main code ownership boundaries for `<project-name>`.

## Recommended Sections

- Application shells
- Domain modules
- Shared UI/components
- Data/service layer
- State layer
- Test ownership

## Fill In

For each module or package family, capture:

- responsibility
- allowed dependencies
- forbidden dependencies
- owner-docs that govern it

## Rule

If a recurring design argument depends on module ownership, write the answer here instead of re-litigating it in chat.
