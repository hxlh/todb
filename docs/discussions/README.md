# Discussion Documents

This directory holds requirement clarification conversations and unresolved question records.

## When to Use

Create a discussion doc when:
- A requirement is ambiguous and needs clarification
- Multiple valid interpretations exist
- Product decisions are needed before implementation
- Trade-offs need to be evaluated before committing to design

## Format

```markdown
# [Topic] Discussion

## Context
What prompted this discussion

## Questions
1. Question one
2. Question two

## Options Considered
### Option A
Pros/cons

### Option B
Pros/cons

## Decision
What was decided (if resolved)

## Open Items
What remains unresolved
```

## Workflow

1. Create discussion doc when ambiguity is discovered
2. Explore options and trade-offs
3. When resolved, synthesize into requirement or design doc
4. Archive or delete the discussion doc (decision lives in the owner doc)
