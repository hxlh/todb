# Requirements Index

Use this directory for implementation-ready requirement synthesis.

Do not dump raw PM chat, copied articles, or prototype notes directly here. Those belong in `docs/input/` first.

## Rule

Only move content here after it is clear enough to drive design or implementation.

This preserves the distinction between raw input and settled requirement intent.

## Suggested Files

- `00-requirement-synthesis-guide.md`
- `product-scope.md`
- `mvp.md`
- `feature-001-<name>.md`
- `2026-05-21-0900-user-management.md`
- `2026-05-21-0900-order-refund-flow.md`

Recommendation:

- use stable names for long-lived baseline files like `product-scope.md` and `mvp.md`
- use dated filenames for one-off requirement synthesis files tied to a specific round of work
